## üß™ Testing with MCP Inspector (Recommended)

### Step 1: Prerequisites

Ensure you have:

* **Java 17 or higher**
* **Maven**
* **MCP Inspector UI**

---

### Step 2: Build the JAR

From the project root directory:

```bash
mvn clean package
```

This will generate a runnable JAR inside the `target/` folder.

---

### Step 3: Open MCP Inspector

Launch MCP Inspector in your browser.

---

### Step 4: Configure MCP Inspector

In MCP Inspector, configure the server connection as follows:

#### Transport Type

```
STDIO
```

#### Command

```
java
```

#### Arguments

```
-jar /absolute/path/to/target/<generated-jar-name>.jar
```

Example:

```
-jar /Users/yourname/projects/mcp-server-stdio/target/mcp-server-stdio-1.0.0.jar
```

> ‚ÑπÔ∏è Always use an **absolute path** to the JAR file.

---

### Step 5: Connect

* Click **Connect** in MCP Inspector
* Status should change to **Connected**

At this point, MCP Inspector has successfully started your server as a child process.

---

### Step 6: Discover Tools

1. Navigate to the **Tools** tab
2. Click **List Tools**

You should see:

* `createTicket`
* `getTicketStatus`

This confirms that the MCP handshake and capability discovery worked correctly.

---

### Step 7: Execute a Tool

Example flow:

* Select **getTicketStatus**
* Provide a username (for example: `ashwani`)
* Click **Run Tool**

The response returned is generated by the MCP server via STDIO.
